## Long English Post

```text
AI coding agents are fast, but they often start coding before the requirement is clear.

That creates rework:
- unclear MVP scope
- missed edge cases
- weak acceptance criteria
- no verification path
- confusing handoff

I built PM Dev Workflow, a universal Claude Code / Codex skill that adds product-manager-grade delivery discipline to coding agents.

Once installed, the agent follows this flow:
clarify requirement -> classify complexity -> surface product/technical risks -> define implementation plan -> build -> self-test -> hand off verification steps.

It works with Claude Code, Codex, or any agent that can read a SKILL.md instruction file.

GitHub: https://github.com/Shangzhen16/pm-dev-workflow
```

## 中文长文案

```text
最近我做了一个 Claude Code / Codex 通用 Skill：PM Dev Workflow。

原因很简单：很多 AI Coding Agent 太容易一上来就写代码。

但真实产品开发里，真正容易出问题的不是“代码写不出来”，而是：

- 需求没澄清
- 用户场景没定义
- MVP 范围没拆清
- 异常状态漏掉
- 没有埋点和验收标准
- 交付后不知道怎么测试

所以我把一个产品经理视角的开发流程做成了 Skill。

装上后，当你说“帮我做一个功能 / 做一个系统 / 改一个需求”时，AI 会自动按这个流程来：

理解需求 -> 判断复杂度 -> 澄清关键问题 -> 给出产品/技术建议 -> 制定开发计划 -> 写代码 -> 自测 -> 给出交付和测试步骤

它适用于 Claude Code、Codex，也适用于任何能读取 SKILL.md 的 AI Coding Agent。

GitHub: https://github.com/Shangzhen16/pm-dev-workflow
```
