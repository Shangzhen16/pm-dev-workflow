# Sample Delivery Output

This is the kind of output the skill should produce for a medium-size request.

## Example Structure

```text
Goal
Scope
Assumptions
Risks
Plan
Files to change
Test plan
Delivery steps
Known limits
```

## Example

- Goal: help users know whether refresh returned new content
- Scope: show success / empty / failure states
- Risks: repeated refresh, inconsistent API response, empty state confusion
- Plan: update PRD, UI, analytics, and verification steps
- Test plan: refresh with new content, refresh with no content, refresh failure

## Why this matters

GitHub visitors can immediately see that the skill produces structured, useful output.
